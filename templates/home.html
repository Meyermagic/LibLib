What is LibLib?

LibLib is a programming "meta-library" site, with the goal of making reusable code more usable. LibLib provides a framework for sharing small, reusable pieces of code, while avoiding the problems associated with many another snippet, recipe, or library sharing sites.

How does it work?

LibLib stores four types of code: algorithms, data structures, test suites, and demonstrations. Algorithms operate on some data structure, and produce some output. Data structures store information in a way that makes it easy to use for some specific application. Test suites verify the correct operation of an algorithm or data structure. Finally, demonstrations show how some simple task may be accomplished.

As an example, let's take a Polygon object. The core of the library would consist of a Polygon data structure. It would implement a few useful method, one of which might be "Point in Polygon". If implemented by a crossing number test, the Point in Polygon method would use another library on LibLib, the algorithm for a ray-line segment intersection test. To verify that it works, a test suite consisting of some pre-computed simple shapes, along with some random ones, would be linked to the Polygon data structure. Finally, the Polygon data structure would have an associated demonstration, showing how it could be used in your own code.
